## 4. Root-cause notes (for the fix pass — NOT fixed here)

### F1 — the 500 "Cannot parse threshold" (CRITICAL)
`serveMarket` → within the 15-min cache TTL but >60 s since the last probe → **PROBE** branch →
`deps.probeLifecycle(id)`. `lib/compute.probeLifecycle` builds a **survival bootstrap config**
(`parse_pattern: '\$(\d+\.?\d*)'`) and calls `fetchEventStatus` → `fetchMarketMeta`, which throws
`Cannot parse threshold` on any leg whose question has no `$X` — i.e. **every binary / categorical /
touch / bucket market**. So `probeLifecycle` is survival-only and 500s for all other shapes whenever
the serve takes the probe path. Why it isn't seen constantly: RESOLVED markets `SERVE_FINAL` (no
probe), recently-probed ones `SERVE_FRESH`, and past-TTL ones `RECOMPUTE` (which DOES classify shape
first). The bug window is **OPEN, non-survival, cached <15 min, last probe >60 s** — exactly where a
freshly-added binary lands a minute after adding. Anthropic (touch) renders only because its cache is
old enough to take the RECOMPUTE path. **The probe must classify shape first (like `computeMarketRecord`
does) instead of assuming a ladder.**

### F2 — Bitcoin not backfilled
On the rail since before this session; detail is fully "collecting" (1/7 velocity, 0 chart points).
Either it was added before `CRON_SECRET` was set, or its backfill failed/never fired. Check
`markets.backfill_status` for this id — it should be `done` with a `backfilled_through` date.

### F3 — implied median "—"
`[data-field="median"]` shows a bare `—` for both ladder markets that don't cross 50% cleanly
(`cap-above` resolved step-CDF; `bitcoin` noisy de-vig). Bug 5 (`impliedMedianLabel`) was supposed to
emit `< $lowest` / `> $highest` (never a bare dash) — either the headline isn't routing through it,
or a different `data-field="median"` element (the SVG median tick) is the one showing `—`. Needs a
look at which node renders the dash.
