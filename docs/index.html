Methodology & provenance

How these numbers are computed (methodology v1.0.0)
  • Arbitrage adjustment: each "above $X" market is a separate book, so raw midpoints can violate monotonicity (negative buckets). We apply volume-weighted isotonic regression (PAVA) to a non-increasing CDF; every metric uses the adjusted curve, and raw_prob is preserved (△ marks an adjusted row). Per-snapshot adjustment size is published.
  • Implied median = adjusted CDF crosses 50% (interpolation). The range is the spread-implied band: the bid and ask curves each crossed at 50% (live only).
  • Implied mean (approx) = bucket-weighted EV; the range is a 3×3 tail-midpoint sensitivity grid (lowest−0.10/0.15/0.20, highest+0.30/0.40/0.60). Assumption-sensitive.
  • 50% confidence band = p25/p75 valuations (adjusted CDF at 0.75 / 0.25).
  • Bucket % = adjusted P(>this)−P(>next); incl. a <$1T bucket; all buckets ≥ 0 and sum to 100% (asserted at build).
  • Confidence = worst of {threshold count, monotonicity adjustments, spread, thin-liquidity breadth, anomalies (stale / closed markets / volume drop)}; reasons shown up top. Backfilled history is price-only → capped at medium.
  • Provenance: every snapshot stores raw inputs + a sha256 you can re-verify (button above). Contract: schema.json · methodology.json · latest.json · API docs.
