# API — `/api/v1/`

A small, static, **public JSON API** for the SpaceX-IPO-market-cap signal. The files under
`docs/api/v1/` are served by GitHub Pages and *are* the product.

- **Base URL:** `https://ilanbenamaro-cyber.github.io/polymarket-tracker/api/v1/`
- **CORS:** every file is served with `Access-Control-Allow-Origin: *` — fetchable from any origin.
- **Refresh cadence:** regenerated by GitHub Actions on each cron tick (≈9:00 AM, 9:30 AM, 4:00 PM ET on weekdays); CDN `Cache-Control: max-age=600` (10 min).
- **Versioning:** breaking schema/formula changes ship under a new path (`/api/v2/`). Within v1, `methodology_version` (semver) signals computation changes; the JSON shape stays stable.
- **Scope:** public Polymarket data only.

## Endpoints

### `GET /api/v1/latest.json`
The full canonical snapshot record — the single source of truth. Contains provenance
(`raw_inputs` + `raw_sha256`) and all derived metrics with a confidence assessment.

```jsonc
{
  "schema_version": "1.0.0",
  "methodology_version": "1.0.0",
  "asset": { "id": "spacex-ipo-market-cap", "name": "SpaceX IPO closing market cap",
             "platform": "polymarket", "market_url": "https://polymarket.com/event/spacex-ipo-closing-market-cap-above",
             "resolves": "2027-12-31" },
  "snapshot": {
    "snapshot_id": "2026-06-05T14:10:01.123Z",
    "fetched_at": "2026-06-05T14:10:01.123Z",
    "source": {
      "provider": "Polymarket",
      "endpoints": ["https://gamma-api.polymarket.com/events?slug=…", "https://clob.polymarket.com/midpoints", "https://clob.polymarket.com/prices"],
      "raw_sha256": "1ebdf8d98876…"
    },
    "raw_inputs": [
      { "token_id": "2449486313…", "threshold": 2, "midpoint": "0.69", "best_bid": "0.68", "best_ask": "0.7", "volume": 764193.186 }
    ],
    "derived": {
      "implied_median": 2.19, "implied_mean": 2.273125,
      "iqr": { "p25": 1.927, "p75": 2.55 }, "total_volume": 5523019.4,
      "confidence": { "tier": "medium", "score": 0.889, "reasons": ["1 monotonicity violation(s)"] },
      "markets": [ { "label": ">$2T", "threshold": 2, "prob": 0.69, "volume": 764193.186, "bucket_prob": 0.2 } ]
    }
  }
}
```

**Verify provenance:** SHA-256 of the canonical JSON of `raw_inputs` (keys ordered
`token_id, threshold, midpoint, best_bid, best_ask, volume`; array sorted ascending by
`threshold`; values exactly as stored) equals `snapshot.source.raw_sha256`. The dashboard's
*verify hash* button does this in-browser via `crypto.subtle`.

### `GET /api/v1/history.json`  *(lean — for charts)*
One small object per UTC day; this is what the dashboard loads for its trend chart.

```jsonc
[
  { "date": "2026-06-05", "implied_median": 2.19, "implied_mean": 2.273125,
    "iqr": { "p25": 1.927, "p75": 2.55 },
    "prob_1_8t": 0.855, "prob_2_0t": 0.69, "prob_2_4t": 0.355,
    "confidence": { "tier": "medium" } }
]
```

### `GET /api/v1/history-full.json`  *(heavy — per-day markets)*
Same days as `history.json` but each entry also carries the full `markets[]`
(`label, threshold, prob, volume, bucket_prob`) and the full `confidence` object. Used for
per-threshold day-over-day deltas and the movers panel. Larger; load only when you need
per-threshold history.

### `GET /api/v1/history.csv`
Flat CSV, one row per day. Columns: `date, implied_median, implied_mean, iqr_p25, iqr_p75,
confidence_tier,` then one column per threshold (`>$1T … >$4T`). Empty cells = that threshold
did not yet trade on that date.

```csv
date,implied_median,implied_mean,iqr_p25,iqr_p75,confidence_tier,>$1T,>$1.2T,…,>$4T
2025-12-12,1.1454545454545455,1.5804999999999998,,,low,0.58,0.47,…,
```

### `GET /api/v1/methodology.json`
Exact copy of [`core/methodology.json`](core/methodology.json): per-metric `method`, `robust`,
`assumptions`, `sensitivity_note`, and a `changelog`. See [METHODOLOGY.md](METHODOLOGY.md).

### `GET /api/v1/snapshots/YYYY-MM-DD.json`
Archived full canonical record for that day (same shape as `latest.json`, including
`raw_inputs`). The immutable audit trail.

## Notes
- All probabilities are `0.0–1.0`; valuations are in **$ trillions**.
- Confidence tiers: `high | medium | low` (see METHODOLOGY.md). Backfilled history is
  price-only → capped at `medium`.
- This API exposes **no** secondary-market / grey-market data.
